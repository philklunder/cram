# meta/ — Decision reasoning

The *why* behind how Cram is built — for a future developer or Claude session with zero context.
This complements (does not replace) the formal numbered decision record in
[`docs/adr/`](../docs/adr/): ADRs freeze the contracts/decisions; these files capture the reasoning,
trade-offs, and downstream constraints in one place.

| Topic | Summary |
|-------|---------|
| [auth-security-posture.md](auth-security-posture.md) | Fail-closed auth defaults, server-side JWT (ES256/JWKS; HS256 dev-only + JWKS required at boot; claim-presence enforced), app-layer ownership (RLS as defense-in-depth), minimal `/healthz`, deferred cost-DoS controls. A **2026-07-06 full-app pass** added a web HTTP-hardening layer (CSP/headers + login anti-enumeration — see web-dashboard.md). Cross-refs ADR 0005–0008. |
| [ios-auth-client.md](ios-auth-client.md) | **Client side of auth.** iOS signs in with the `supabase-swift` SDK (Keychain + auto-refresh kept as default), sends `Authorization: Bearer <jwt>` (drops `X-Cram-Secret`); token fetched per-request, never cached; auth gate only when configured; HTTPS-only bearer, enumeration-safe errors, local-scope sign-out fallback. Cross-refs ADR 0007/0008. |
| [data-layer-and-sync.md](data-layer-and-sync.md) | **Backend** Phase 3 structure: single owner-scoped repository, parent-steal/id-squat defenses, compound keyset delta cursor, app-code soft-delete cascade, router factory. Cross-refs ADR 0007/0008. |
| [exam-level.md](exam-level.md) | **Subject → Exam → Cards/Quiz (2026-07-07).** New owned+syncable `exams` table (migration `0005`, RLS + `auth.users` FK like every domain table) as a level between subject and card. `cards.exam_id`/`quizzes.exam_id` are **nullable `ON DELETE SET NULL`** — a card outlives its exam and falls back to the subject's **"General"** bucket (mirrors card→source); exam is deliberately **omitted from the soft-delete cascade** so deleting an exam never tombstones its cards. Optional owned-parent check on `exam_id`; `exam_date` per exam drives its countdown + SM-2 compression; `/v1/generate` gains an optional `exam_id`. Web adds a reusable a11y `Modal` + `ExamFormModal`/`SubjectFormModal` and fetches `exams` in the subject bundle. Contract mirrors a future iOS `Exam @Model`. **2026-07-07:** `subject.exam_date` retired (migration `0006`) now that exams own the date — web derives a subject countdown from its soonest upcoming exam; iOS reconciliation deferred to Phase 5. **2026-07-08:** grades link to exams via a nullable `grade_entries.exam_id` (`SET NULL`, migration `0007`, applied to prod); a graded exam is **derived-archived** into a web **"Past exams"** section (no status flag — graded ⇔ done, reversible) and its cards leave active revision; the Grades logger becomes a guided **Subject→Exam→grade** and "New subject" moves out of Grades; iOS gets `GradeEntry.examId` for sync parity (no UI). Cross-refs ADR 0007/0008. |
| [progress-model.md](progress-model.md) | **Practice vs assessment (2026-07-09).** The app's measurement contract: **Flashcards + Quizzes are practice** (record study time only — no SM-2, no attempts), and a **Review** is the sole assessment (recall phase → `PATCH /v1/cards` + `/v1/review-logs`; test phase → `POST /v1/attempts`). Those two writes are the only inputs to **readiness** = `0.5·cardMastery + 0.3·quizAccuracy + 0.2·coverage`, with a missing signal's weight redistributed and `untested` a first-class state (never a 0%). A `Question` has no SM-2 state and no FK to a `Card` (shared `topic` string only), so quiz answers deliberately **never touch a card's schedule** — the signals stay separate and combine only at the readiness layer. Practice writes no attempts (rather than tagged ones) because `attempts` has no source column and tagging needs a migration + iOS parity. One definition app-wide (`lib/readiness.ts`); `examReadiness()` falls back to the subject when an exam has no material of its own (the upload defaults to "General"). Cross-refs ADR 0002/0004, exam-level.md. |
| [ios-sync-client.md](ios-sync-client.md) | **Client side of sync (v0.5 Phase 5).** Offline-first, last-writer-wins, push-then-pull; sync metadata + `touch()`/`softDelete()` on the models; tombstone deletes; generated decks adopt server ids (de-dup); per-user cursors + user-switch wipe; trailing throttle + `Retry-After` 429 backoff. File-bytes not synced. Cross-refs ADR 0007/0008. |
| [cost-controls.md](cost-controls.md) | Phase 4: Postgres-backed per-caller rate limit + token-based per-user/global daily spend cap; why metering commits before persistence (a spend-cap-bypass fix); why the rate limit guards the DB not the wallet (size it for the sync client); generate degrades (keeps the deck) on a Storage hiccup; prod-config guard; **`rate_limit_buckets` lazy in-process prune (2026-07-06)**. Cross-refs ADR 0009. |
| [edge-and-budget-backstops.md](edge-and-budget-backstops.md) | Security layers the app can't enforce: reverse proxy (body/flood ingestion) + Anthropic Console hard cap (spend) as mandatory out-of-app backstops; chosen prod cap values; floors-vs-lockfile. From the 2026-06-18 pre-deploy audit. |
| [deployment.md](deployment.md) | **LIVE on Railway (2026-06-19).** Host = Railway Hobby; the M1 reverse proxy (nginx) is baked into the container in front of uvicorn so the body/flood cap holds portably; why Vercel/Render were rejected for the backend; deploy packaging in `backend/`; go-live boot bugs (Railway quote-stripping, nginx duplicate-directive). Cross-refs ADR 0009. |
| [web-dashboard.md](web-dashboard.md) | **Web client (v0.6+).** Next.js "study desk": browse/upload/progress, **plus quiz-taking + SRS review + a Grades section (2026-06-30):** MC graded client-side → `/v1/attempts`, short-answer → paid `/v1/grade`; review runs a full SM-2-parity port of the iOS scheduler (`src/lib/srs/`, pinned by a 34-vector Vitest suite) and writes back `PATCH /v1/cards` + `/v1/review-logs`; **Grades tab** does `POST`/`DELETE /v1/grade-entries` + `PATCH /v1/subjects` (current-vs-target, Pass/Fail), reusing `grade-strength.ts` so the grade math has one source of truth. Direct browser→backend Bearer JWT (mirrors iOS, not a BFF); `@supabase/ssr` cookie auth + `getUser()` server gate; client-side fetch+filter over the delta endpoints. A **2026-07-02 visual redesign** adds per-subject color identity (deterministic, CSS-variable driven so it survives Tailwind JIT) + a `motion` interaction layer (sliding tabs, spring entrances), with `PRODUCT.md`/`DESIGN.md` as the design source of truth. A **2026-07-03 pass** adds full **light/dark theming** (semantic RGB-triple token layer, `darkMode: class`, no-flash toggle) and a **Linear-inspired restraint retune** (neutral surfaces, per-subject color demoted to one quiet tile, calmer state-driven motion; entrances no longer gate visibility). A **2026-07-04 bolder pass** swaps the `brand` accent cobalt → electric-violet (a one-file token retune that re-hues the app) and rebuilds the **login** to a reference mock. A **2026-07-06 pass** completes it: the app moves to a **left-sidebar shell** (`components/shell/` + routed `(app)/` destinations, `AppNav` deleted), all **8 surfaces matched to owner reference mocks** (verified light+dark via the `preview/` harness), a new append-only **`study_sessions`** owned resource + migration `0004` (applied live) feeding the weekly-activity chart, and **`DESIGN.md` rewritten** to the violet sidebar system (`/security-review` clean). A **display grading-scale preference** (device-local `useDisplayScale`; Settings picker) then lets aggregate grade numbers (overall/subject averages, Progress trend) render as a chosen scale, e.g. Swiss 6.0–1.0, via an inverse of the normalize-to-% layer — presentation only, entries + weights + pass-rate stay native/%. Forced the backend's env-driven `CRAM_CORS_ORIGINS` allowlist (default closed, credentials off). A **2026-07-06 security pass** added HTTP security headers + a CSP (`frame-ancestors 'none'`; `connect-src` derived from the `NEXT_PUBLIC_*` env) via `next.config.mjs`, and hardened the login form against account enumeration + weak sign-up passwords. A **2026-07-07 pass** gates the HTTPS-forcing headers (HSTS + `upgrade-insecure-requests`, plus a dev-only `'unsafe-eval'`) on `NODE_ENV` so localhost dev serves over plain http while prod hardening is unchanged, alongside a presentational polish pass (named greeting, richer Settings/AI-Decks/Quizzes, server-side preview-harness slug). A **2026-07-08 IA cleanup** makes the thin "funnel" hubs self-contained via a shared **Subject→Exam scope picker** (`lib/scope.ts`): Quizzes/Flashcards act in place (Flashcards gains a schedule-neutral "cram" flip-through), **all add-material deep-links to AI Decks** (inline form deleted), **Grades consolidates to one editor** on the Grades page (subject view read-only), and the **Calendar** drops fabricated suggestions for real exams + localStorage study blocks. A **2026-07-08 pass** rebuilds the **Quizzes + Review hub pickers** to owner reference mocks (two-step Subject→Exam selector; a self-contained inline-SVG illustration on Review; stats as client-side heuristics — accuracy from 7-day `review_logs` ratings, estimated-time from per-card constants — no new backend), and adds a **device-local Review-settings store** (`lib/reviewSettings.ts`, same `localStorage`+`useSyncExternalStore` pattern as `useDisplayScale`) whose session-size + card-order knobs are wired straight into `ReviewSession.buildQueue` (`order`/`limit` props) — deliberately not backend-synced. Cross-refs ADR 0005/0006/0007/0008. |
